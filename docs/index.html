Features

Alles in einer Anwendung

Von der Bildannotation über das Modell-Training bis zum produktionsreifen Edge-Deployment.

🏷

Bild-Labeling & ROIs

ROI-Editor mit Rechteck, Ellipse und Polygon. Multi-Label-Modus, Undo/Redo-Stack, Tastenkürzel für maximale Geschwindigkeit.

🧠

Modell-Training

ResNet-18/50, MobileNetV2, EfficientNet-B0. GPU/CPU/MPS. Early Stopping, Mixed Precision, WeightedRandomSampler.

📷

Live-Anomalieerkennung

Conv-Autoencoder lernt auf Normalframes. Grad-CAM Heatmap, Bounding-Box, Alarm-Pause, False-Positive-Markierung.

📹

Multi-Kamera

Bis zu 9 Kanäle gleichzeitig im 2×2-Grid mit automatischer Paginierung. Alarm-JPEG-Saving und per-Kanal REST-API.

⚙

Hyperparameter-Suche

Optuna-Integration für Klassifikation und Anomalie-Autoencoder. Bestes Ergebnis direkt in die UI übernehmen.

🔧

Kamera-Einstellungen

Live-Sliders für Helligkeit, Kontrast, Sättigung. Vorverarbeitungsfilter: Graustufen, Canny, Sobel, Laplacian.

🌐

Fleet & Edge

Zentrale Überwachung mehrerer Monitor-Instanzen. Docker-Deployment-Generator. ONNX INT8, Apple CoreML.

📊

REST API & Protokolle

HTTP-Server mit Live-Dashboard. MQTT, OPC-UA, Modbus TCP, E-Mail- und Webhook-Alarmierung.

🔗

Open Source

MIT Lizenz. Keine versteckten Abhängigkeiten. Vollständige Quelltexte auf GitHub. Pull Requests willkommen.

Quickstart

In 60 Sekunden startklar

PictureStudio benötigt Python 3.10+ und keine Root-Rechte.

Installation 
# Repository klonen
git clone https://github.com/oOMaikOo/PictureStudio
cd PictureStudio

# Abhängigkeiten installieren
pip install -r requirements.txt

# PictureStudio starten
python main.py
Monitor-Client (standalone) 
# Modell laden und Kamera starten
python monitor.py --model models/anomalie.pth

# ONNX ohne PyTorch (Edge-Deployment)
python monitor.py --model models/anomalie.onnx

# Headless (kein Fenster, nur CSV + Logs)
python monitor.py --model models/linie1.pt \
        --headless --output /var/log/anomalien

# Web-Setup-Assistent
python monitor.py --setup
Optionale Abhängigkeiten 
# Excel-Export
pip install openpyxl

# SSH-Ferntraining
pip install paramiko

# MQTT-Alarm
pip install paho-mqtt

# Hyperparameter-Suche
pip install optuna

# Kalibrierung & INT8-Export
pip install scipy onnxruntime

# Duplikaterkennung
pip install imagehash

# CoreML (macOS)
pip install coremltools
Workflow

Drei Schritte zum Produktivmodell

PictureStudio begleitet den kompletten ML-Lifecycle — von den Rohdaten bis zum laufenden Edge-System.

①

Daten & Labeling

Bilder laden, Labels definieren, ROIs zeichnen. Multi-Label-Modus für komplexe Szenarien. Undo/Redo für fehlerfreie Annotation. Datensatz-Analyse auf Duplikate und Klassenungleichgewicht.

②

Training & Optimierung

ResNet, MobileNet oder EfficientNet lokal oder per SSH auf GPU-Servern trainieren. Echtzeit-Kurven, Early Stopping, Optuna-Hyperparameter-Suche. Modellbibliothek mit Accuracy/F1-Vergleich.

③

Deploy & Monitoring

Live-Monitoring mit bis zu 9 Kamerakanälen. Anomalie-Autoencoder mit Heatmap und Alarm. Export als ONNX INT8, CoreML oder Docker. Fleet-Management für mehrere Edge-Geräte.
